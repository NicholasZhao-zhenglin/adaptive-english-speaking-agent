# 自适应英语口语 Agent 技术方案

> 本文对应独立仓库 `adaptive-english-speaking-agent`。该模块已从
> `personal-assistant` 的应用入口、前端 DOM 和运行数据中解耦；原仓库中的实现仍然保留。

## 1. 背景与目标

原英语模块能够定时生成表达和互动练习，但内容选择主要由天数和随机历史抽样决定，练习结果没有进入下一次学习决策。升级目标是建立完整闭环：

```text
学习目标 → 每日规划 → 内容生成 → 互动练习 → 结果评估
    ↑                                         ↓
    └──────── 掌握度记忆 ← 错因诊断 ←───────┘
```

本版本暂不接入语音。Agent 的核心能力是持久化状态、自主规划、工具执行、结果评估和动态调整，而不是是否使用语音或是否拆成多个 Agent。

## 2. 架构设计

### 2.1 职责分层

- `english_learning_agent.py`：管理学习画像、表达掌握度、间隔复习、每日计划与诊断，是决策层。
- `english_assistant.py`：执行内容与题目生成、历史去重和缓存，是内容工具层。
- `backend/app.py`：只暴露英语画像、仪表盘、内容和练习结果接口，并托管独立前端。
- `frontend/`：展示决策理由、编辑画像、完成练习并回传结果。
- `backend/generate_daily.py`：供任务计划调用的每日内容生成入口。

采用“确定性调度器 + LLM 内容生成”的混合方案：调度器决定学什么、学多少、何时复习和题型比例；LLM 在约束内生成自然表达和题目。这样可以降低模型随机性导致的计划漂移，并使核心策略可测试、可解释。

### 2.2 数据文件

| 文件 | 内容 | 关键字段 |
|---|---|---|
| `english_profile.json` | 学习者画像 | `goals`、`level`、`daily_minutes`、`interests` |
| `english_learning_state.json` | 表达级长期记忆 | `mastery`、`streak`、`next_review_at`、`error_counts` |
| `english_sessions.json` | 最近练习会话 | `accuracy`、`attempts`、`completed_at` |
| `english_memory.json` | 已生成表达历史 | `day_count`、`history` |

文件均位于 `backend/data/`，由 `.gitignore` 排除，避免把个人学习记录提交到仓库。

## 3. 核心算法

### 3.1 掌握度更新

每个表达维护 0—1 的 `mastery`：

- 答对：`mastery = min(1, old × 0.65 + 0.35)`，连续答对次数加一。
- 答错：`mastery = old × 0.6`，连续次数归零，并累计错因。

这种指数更新让近期表现权重更高，同时避免一次答对就被判定为完全掌握。它不是语言学习效果的科学量表，而是当前个人项目中可解释、可迭代的启发式指标。

### 3.2 间隔复习

答对后的复习间隔依次为 1、3、7、14、30 天；答错后当天重新进入到期队列。到期内容按以下优先级排序：

1. 更早到期；
2. 掌握度更低；
3. 历史错误更多。

该策略借鉴间隔复习思想，但没有声称实现完整 SM-2。后续可以用真实学习数据校准间隔和权重。

### 3.3 每日计划

Agent 使用最近 5 次会话的平均正确率动态决定新旧比例：

| 近期正确率 | 新表达 | 复习表达 | 策略 |
|---|---:|---:|---|
| `< 60%` | 4 | 8 | 降低新内容，强化复习 |
| `60%—84%` 或数据不足 | 6 | 6 | 均衡训练 |
| `≥ 85%` | 8 | 4 | 增加新内容 |

目标主题从用户画像中轮换，内容生成提示还包含 CEFR 水平和兴趣场景。

### 3.4 错因诊断与题型调度

练习错误分为含义辨认、主动回忆、场景运用和表达配对。累计最多的错因决定下一轮题型配比：

- 含义辨认弱：增加选择题；
- 主动回忆弱：增加填空题；
- 场景运用弱：增加场景应用题；
- 配对弱：增加连线题。

前端提交的是表达级结果而非只有总分，因此同一练习可以更新多个表达的长期状态。

## 4. API 契约

| 接口 | 方法 | 说明 |
|---|---|---|
| `/api/english/profile` | GET / PUT | 读取或更新学习画像 |
| `/api/english/dashboard` | GET | 返回今日计划、统计、薄弱项和到期内容 |
| `/api/english/attempts` | POST | 记录表达级练习结果并返回更新后的计划 |
| `/api/english/today` | GET | 按当前计划生成每日新表达 |
| `/api/english/practice/<day>` | GET | 按薄弱项配比生成练习 |

`POST /api/english/attempts` 示例：

```json
{
  "day": 3,
  "attempts": [
    {
      "expression": "That makes sense",
      "meaning": "有道理",
      "correct": false,
      "exercise_type": "scenario_choice",
      "error_type": "context"
    }
  ]
}
```

## 5. 可靠性与安全边界

- JSON 写入采用临时文件后 `os.replace`，降低进程中断导致半文件的风险。
- 使用进程内可重入锁保护单次文件读写；当前仍以单进程、单用户为运行假设。
- 画像字段限制长度、枚举和值域，避免无限扩张提示词。
- 练习结果每次最多 100 条；内容补位最多重试 5 次，避免异常输入或重复输出造成无界消耗。
- 所有前端动态内容继续经过 `escapeHtml`。
- 英语 API 使用相对 URL，不绑定固定端口，便于本地验证和部署。
- 前端在解析 API 前校验 `Content-Type`；旧服务缺少新路由并返回 HTML 时，展示“重启本地助手”的可操作提示，而不是暴露 JSON 解析异常。
- 启动日志只记录启动阶段和结果，不记录 API Key 等密钥值。

## 6. 测试与验收

单元与接口测试覆盖：

- 画像标准化和持久化；
- 答对/答错后的掌握度、连续次数和复习日期；
- 低正确率时的新旧内容比例；
- 到期低掌握表达的排序；
- 错因对应的动态题型配比；
- 练习结果 API 返回更新后的仪表盘；
- 仪表盘路由返回 JSON 契约，前端拒绝 HTML 等非 JSON 响应；
- 内容生成遵循 Agent 的数量和主题计划。

浏览器验收覆盖桌面和 320px 宽度：页面无横向溢出，计划区正确响应，画像表单可打开和取消，英语内容通过相对 API 正常加载。

## 7. 已知限制与后续方向

- 掌握度公式和复习间隔是启发式规则，需要用长期正确率和复习保持率校准。
- JSON 适合个人单进程应用；若引入多进程或多用户，应迁移 SQLite/PostgreSQL 并使用事务。
- 当前评估来自客观题，不能衡量自由口语的流利度和发音。
- 当前题目由 LLM 输出 JSON，应进一步加入完整结构校验和生成质量离线评测集。
